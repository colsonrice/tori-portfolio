Case Study

Enterprise Agentic Search

Role Lead Product Designer
Team 3 designers, 6 engineers, 2 PMs
Timeline 2024 – 2025
Overview

From keyword matching to platform-wide intelligence.

Enterprise search touched every object, file, and setting in the platform — but returned noise instead of answers. I led design for the setup and configuration layer of a new agentic search product, focusing on how admins define search scope, configure ranking rules, and map intent categories across the platform's data model.

The Challenge

A search bar sitting on top of a data model nobody could navigate.

Salesforce's data model spans thousands of object types, files, knowledge articles, and admin settings. Search returned unranked results across all of them with no understanding of what the user actually needed. Users spent more time filtering than finding — and once they found something, they had to leave search entirely to act on it.

Process

From retrieval to resolution.

1

Discovery

Mapped how admins currently configure search behavior across objects, fields, and permissions. Identified the technical constraints that made search setup opaque — scattered settings, no preview of results, and no feedback loop on ranking quality.

2

Architecture

Designed the admin-facing configuration surface: search scope definition (which objects, fields, and records to index), intent-mapping rules, and ranking weight controls — giving admins a clear model for how search behaves without exposing the underlying ML pipeline.

3

Design

Created the setup flow including a live preview mode that lets admins test queries against their configuration before publishing. Designed the intent category editor and result-type mapping interface.

4

Validation

Tested configuration flows with admins across org sizes. Iterated on the mental model for ranking controls — balancing technical precision with approachability for non-technical admins.

Results

Admin-ready search configuration.
